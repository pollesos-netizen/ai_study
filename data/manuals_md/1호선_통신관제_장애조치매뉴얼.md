## 01 관제전화설비

### 01-01 관제전화 녹음설비 백업 오류 조치

1. **CallSaverFTP SM 1.5.1 #2 → RecToFileDB 실행**

2. **WAV, NFO 파일 재등록 → 해당일자 수정 검색 실행**

3. **파일검색 완료 후 → Add Proc 실행** (백업완료 후 프로그램 재기동)

4. **CallSaver 설정 실행 → 카드등록 실행**

> ※ 백업오류 현상 방지 월 1회 서버 제어PC 재부팅 실행  
> (주/예비 동작 확인 후 개별 실행 – 주/예비 동시 하지 말것)

---

### 01-02 관제전화 녹음설비 장애로 저장카드 리셋조치

1. **관리메뉴 더블 클릭 → 카드재부팅 항목 생성**

2. **카드 IP 확인**

3. **카드 재부팅 실행**
   - Yes → 해당 카드 1개 재부팅
   - No → 전체카드 재부팅 항목으로 이동

4. **Yes 실행 → 사용중인 모든 카드 재부팅**

> ※ 저장카드 반기 1회 재부팅 실행  
> (주/예비 동작 확인 후 개별 실행 – 주/예비 동시 하지 말 것)

---

### 01-03 관제전화 장애로 DCU 카드 원격 리셋조치

1. **상단 우측 부분을 마우스로 더블클릭**

2. **소프트웨어적인 MCU카드 초기화 버튼 생성**

> ※ 업체에서 임시적으로 만들어놓은 기능입니다.  
> 'MCU 초기화' 클릭 시 해당 조작반의 DCU카드가 리셋됩니다.  
> (카드 탈실장과 동일)

## 02 전송설비

### 02-01 NMS 감시 불량

**■ Alarm 확인**
- NMS 감시불량
- 전체노드 Node offline

**■ 조치내용**
- 파트 NMS LOG OUT
- SOA 접속 확인
- 컨넥터 접속 확인
- SOA 재기동

**■ 참고사항**
- 디지털전송설비 운용매뉴얼

**작업 절차 (흐름도)**

```
장애발생
  ↓
감시불량 확인
  ↓
파트 LOG OUT
  ↓ [Yes: SOA 접속 가능?]
  ├─ Yes → [컨넥터 접속 확인?]
  │           ├─ Yes → [SOA 재기동?]
  │           │           ├─ Yes → 동작상태 확인 → 복구완료
  │           │           └─ No  → SOA 점검
  │           └─ No  → SOA 점검
  └─ No  → SOA 점검
              ↓
          동작상태 확인
              ↓
          복구완료
```

---

**■ 참고사항**
- 디지털전송설비 운용매뉴얼

**작업 절차 (흐름도)**

```
장애발생
  ↓
감시불량 확인
  ↓
통신 확인(PING)
  ├─ Yes → CU 리셋
  │           ├─ Yes → 동작상태 확인 → 복구완료
  │           └─ No  → CU 절체
  │                       ├─ Yes → 동작상태 확인 → 복구완료
```

## 04 열차무선설비

### 04-01 1호선 열차무선 MCE 알람 조치내용

**증상**
- 프로그램 실행 시: 좌측 하단에 COMMUNICATION ERROR 발생
- 열차무선 통화 서비스에는 이상없음
- 현장파트(기지, 북부, 남부) AME, 관제 MCE에서 기지국 감시 및 제어가 안됨

**조치사항**
- 422/232K, 422IF2, 232IF 예비품 카드로 교체
- FRAME B 전원 RESET
- MCE 예비품PC로 교체
- 연결케이블 결선 확인 및 재접속 (알람 미복구)
- 주장치 FRAME(B) 커넥터 교체 및 5회선 재납땜
- 프로그램 재실행 → 알람확인

**CCE B FRAME ↔ MCE 케이블 결선도 (CNB34 ↔ 통신관제 MCE)**

| CNB34(25PIN) PIN NO | SIGNAL | 연결회선(5회선) | MCE(9PIN) PIN NO | SIGNAL |
|---------------------|--------|----------------|-------------------|--------|
| 1 | FG | | 4, 5 | RS, CS |
| 2 | SD3 | 빨강 | 2 | RD |
| 3 | RD3 | 흰색 | 3 | SD |
| 4 | RS3 | 검정 | 1 | CD |
| 5 | CS3 | | 1 | CD |
| 6 | | | | |
| 7 | SG | 주황 | 5 | CD |
| 8 | CD3 | 노랑 | 7, 8 | RS, CS |

> ※ CNB34케이블 커넥터 4, 5번 PIN 납땜으로 같이 연결됨  
> (케이블 연결은 검정색으로 4번 PIN에만 연결)

---

### 04-02 열차무선 장애에 따른 주요부 기지국절체

1. **메뉴창에서 CONTROL 선택 → BS CONTROL 선택**

2. **기지국 중에서 절체를 원하는 기지국을 더블 클릭**

3. **CPU상태가 CPU1인 경우 CPU2로 절체 시 → 18번 항목의 CPU2 선택 후 더블 클릭**

4. **OK 버튼 클릭**

---

### 04-03 열차무선 장애에 따른 주요부 송·수신유닛 절체

1. **메뉴창에서 CONTROL 선택 → BS CONTROL 선택**

2. **기지국 중에서 절체를 원하는 기지국을 더블 클릭**

3. **송·수신 UNIT 상태를 확인하고 절체하고자 하는 유닛 선택 (1~4, 9~12)**

4. **OK 버튼 클릭**

## 08 전기시계설비

### 08-01 전기시계 GPS CLOCK 수신 불량 및 부모장치 적색 알람

**증상**
- GPS CLOCK 수신 불량으로 LOCK 램프 OFF 및 시각 비정상

**작업 절차 (흐름도)**

```
MAIN 전원공급 정상여부 확인
  ├─ Y → 전원램프 이상 무
  └─ (계속)
      ↓
  시각 DATA 출력 확인 (30초간격)
      ↓ N
  본사 건물 옥상 GPS 수신장치 리셋
      ├─ Y → 수신 시각 정상화
      └─ N
          ↓
      Main Clock 예비품 교체
          ↓
      수신 시각 확인 및 알람 확인
          ├─ Y → 조치 완료 (시각 정상 및 알람 해소)
          └─ N → GPS PCB 및 SMPS(전원공급장치) 등 해당부품 교체
                      ↓ Y
                  조치 완료 (시각 정상 및 알람 해소)
```
