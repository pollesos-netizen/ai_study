# C/S/O 판정 가이드 (검수자용 rubric)

근거: `생성형 AI 활용을 위한 데이터 보안 가이드라인 v1.0` (슬라이드 8·9) + 사내 위험도 기준
대상: 기술 검사표준·장애조치 매뉴얼의 서술형 문장 (AI 후보)

---

## 핵심 판정 질문 (순서대로)

문장 하나를 볼 때, 아래 질문을 **위에서부터** 던진다. 처음 "예"가 나오는 곳이 등급이다.

### Q1. 이 절차/정보가 외부에 노출되면 공격자가 시스템을 멈추거나, 우회하거나, 취약점을 알 수 있는가?
→ **예: C급 (기밀)**

가이드라인 C급 정의와 직결: "사이버 공격에 직접 악용 가능한 기술 정보", "시스템 우회 방법 등 외부 유출 시 위험을 직접 키우는 상세 매뉴얼".

C급 판정 신호:
- 시스템·장비를 **무력화/우회**하는 조작: BYPASS 취급, 보호장치 해제, 인터록 해제, 강제 절체
- 시스템에 **직접 침투·제어**하는 방법: 설정파일 로딩(CLI 명령), 원격 접속+계정, shutdown/reboot 절차, 카드 리셋 상세 순서
- **보호계전 설정값/정정치** (트립값, 동작 임계값) — 알면 보호체계 무력화 가능
- **세부 연계 구조/우회 방안** — 시스템 간 연결을 끊거나 돌리는 방법
- 절체 **우선순위·조건** (Lockout〉강제〉자동〉수동 같은 제어 위계)

판정 보조 질문: *"이 문장을 공격자가 손에 넣으면, 시스템 정지·오동작·침투에 실제로 써먹을 수 있는가?"* → 예면 C.

### Q2. 개인정보, 또는 계약·평가·인사·내부 의사결정 같은 업무상 민감 정보인가? 혹은 노출돼도 직접 공격엔 못 쓰지만 대외비로 둘 내부 업무 정보(작업 순서 등)인가?
→ **예: S급 (민감)**

가이드라인 S급 정의: 개인정보(성명·연락처·사번·내선), 계약·평가·협상, 인사기록, 장애 원인분석 보고서.
**+ 사내 확장 기준:** 조작 절차라도 **단순 작업 순서**여서 공격 악용도는 낮지만 내부 운영 정보로 보호할 것.

S급 판정 신호:
- 개인정보·연락 정보: 성명, 사번, 내선, 연락처, 비상연락망, 통화기록
- 업무 행정: 근무조 편성, 업무분장, 인수인계, 승인·결재
- 계약·평가: 단가, 원가, 입찰 평가, 협상 전략
- **정형 작업 순서**: 정해진 절차에 따른 계통 작업 순서(예: 연락수전 순서, 차단기 순차투입 순서) — 조작이지만 "공격 악용"보다 "내부 업무 정보" 성격

판정 보조 질문: *"이건 공격 도구라기보다, 사람·거래·정형 업무에 관한 내부 정보인가?"* → 예면 S.

### Q3. 단순 점검·확인·상태 표시·측정 행위인가? (조작도 민감정보도 아님)
→ **예: O급 (공개)**

O급 판정 신호:
- 단순 확인/점검: "LED 표시상태를 확인한다", "동작상태 확인", "상태변화 확인"
- 측정·기록: "온도 측정", "전압 확인", "MIS 전산입력"
- 일반 설비 명칭·일반 동작 서술 (조작 방법이 아닌)

판정 보조 질문: *"이건 그냥 '봤다/쟀다/정상이다' 수준인가?"* → 예면 O.

## C vs S 경계 (가장 헷갈리는 지점)

둘 다 "조작 절차"일 때 가르는 단 하나의 기준:

| | C급 | S급 |
|---|---|---|
| 핵심 | 공격 악용 가능 (시스템 정지·우회·취약점) | 단순 작업 순서 / 내부 업무 정보 |
| 예 | "UPS 수동 BYPASS 취급", "설정파일 CLI 로딩", "절체 우선순위" | "연락수전(임학→작전) 순차투입", "차단기 순차투입 순서" |
| 질문 | 이걸로 공격할 수 있나? | 그냥 정해진 일 순서인가? |

> 차이의 직관: **BYPASS 취급**은 보호장치를 *무력화*하는 방법이라 C. **연락수전 순서**는 정전 시 *정해진 절차대로* 전원을 돌리는 작업 순서라 S. 전자는 "시스템을 깨는 법", 후자는 "규정대로 일하는 법".

## 경계 사례는 memo에 기록

판정이 흔들린 문장은 `userGrade`를 정하되 `memo`에 한 줄 근거를 남긴다.
예: "조작이지만 정형 순서라 S", "확인 행위지만 우회경로 포함이라 C".
→ 이 기록들이 모델 학습 시 '경계 사례' 데이터가 되고, rubric을 다듬는 근거가 된다.
